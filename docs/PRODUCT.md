## 1. 产品一句话

**Kaika：一个本地阅读 App，同时支持漫画（页图）与图书（reflow）两种阅读引擎。**

- 安装包、图标、名字、默认设定 **统一为 Kaika**。
- 用户在同一 App 内混排管理漫画与图书，可通过「类型」筛选。
- 导入 **CBZ / ZIP / EPUB**；EPUB 按内容自动识别为页图漫画或正文图书。
- 共享工程底座：进度模型、导入范式、不透明 locator、书库管理、桌面 chrome、controller 分层。

## 2. 单 App 内共享

### 2.1 统一拥有

| 维度 | 策略 |
|------|------|
| 品牌名 / 图标 / 启动 / 商店页 | **Kaika** |
| Bundle ID / 包名 | `com.kaika.reader` |
| 本地数据目录 | 沿用已有 support root + `app_library`，避免已有数据丢失 |
| 默认阅读主题 | 漫画深灰 / 图书纸白（分别可调） |
| 导入白名单 | CBZ、ZIP、EPUB |
| 根导航与空态 | 中性「书库/书架/导入」语境 |

### 2.2 引擎分叉

| 引擎 | 内容 | 阅读器 |
|------|------|--------|
| 页图 session | CBZ/ZIP、页图/固定版式 EPUB | 漫画阅读器（四模式/方向/主题） |
| reflow spike | 正文 EPUB（纯文本） | 图书阅读器（分节/字号/行距/主题） |

共享：本地文件、content-hash 去重、不透明 locator 进度约定、控制器分层、drift 表范式、设计 token。

## 3. 单 App 内导航

App 仍是三 Tab（宽屏 Rail / 窄屏 BottomBar）：

| Tab | 职责 |
|-----|------|
| **书架** | 继续阅读、最近、我的书架（钉选） |
| **书库** | 全部单本；搜索 / 筛选 / 排序 / 管理；**书单**与**合集**为二级 |
| **设置** | 外观、关于；阅读偏好只在对应阅读器内调整 |

- **搜索**、**书单**、**合集** 宿主在书库侧，**不是**第四根 Tab。
- 书库内通过「类型：全部 / 漫画 / 图书」筛选，取代品牌身份切换。

## 5. 用户路径

1. **首次**：导入 CBZ/ZIP/EPUB → 书库 → 打开阅读 → 书架继续读  
2. **续读**：书架 → 恢复进度  
3. **查找**：书库搜索 / 筛选 / 排序 → 打开  
4. **整理**：删除、上架；书单（队列）；合集（系列盒）

---

## 6. 条目生命周期

```text
导入（CBZ/ZIP/EPUB）→ 书库
  → 打开 → lastOpenedAt + progress → 书架继续/最近
  → 上架 → 我的书架（钉选）
  → 书单 → 清单成员
  → 合集 → 合集成员
  → 删除 → 级联清理进度/成员关系
```
