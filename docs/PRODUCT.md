## 1. 产品一句话

**同一套技术底座，两个独立品牌的本地阅读 App：**

| 代号 | 定位（显示名待定） | 内容重心 |
|------|--------------------|----------|
| **comic** | 漫画 / 页图式阅读产品 | CBZ/ZIP 等图包，全屏页模式 |
| **book** | 图书 / 文字阅读产品 | EPUB 等 reflow（及后续 TXT/MD…） |

- 商店、图标、名字、默认设定、导入格式、空态文案 **分开**。  
- **不**做成一个 App 里「漫画 | 图书」双分段；**不**要求用户在一个壳里混用两种身份。  
- 共享的是工程与能力（进度模型、导入范式、chrome 语言可选复用），不是用户心智上的「一个 Kaika」。

旧名「Kaika」可作组织/仓库名；**上架名按品牌分别命名**（待定，文档里用 comic / book 代号）。

## 2. 双产品 vs 共享

### 2.1 每个 App 各自拥有

| 维度 | comic | book |
|------|-------|------|
| 品牌名 / 图标 / 启动 / 商店页 | 独立 | 独立 |
| Bundle ID / 包名 | 独立 | 独立 |
| 默认阅读主题 | 深灰等沉浸 | 纸白等正文向（待 book spec） |
| 默认阅读控件 | 四模式 / 方向 | 字号 / 行距 / 目录等 |
| 导入白名单 | 图包格式为主 | 图书格式为主 |
| 根导航文案与空态 | 漫画语境 | 图书语境 |
| 本地数据目录 | **隔离**（默认） | **隔离** |
| 发版节奏 | 可独立 | 可独立 |

### 2.2 可共享（工程 / 体验语言，非强制同一皮肤）

- 本地文件、content-hash 去重思路  
- 不透明 locator 进度约定  
- 控制器分层、drift 表范式  
- 轻玻璃中性 **设计 token 体系**（各品牌换强调色与默认阅读主题）  
- 书架「继续阅读」、书库网格、搜索 / 书单等 **交互模式**（文案与入口按品牌裁剪）

### 2.3 明确不做

- 一个安装包内切换「我是漫画用户 / 图书用户」  
- 两 App 默认同库、云同步账号打通（远景另议，现阶段不做）  
- 迁移旧 Swift kaika 用户数据  

---

## 3. 单 App 内导航（两套产品结构相同、内容不同）

每个 App 仍是三 Tab（宽屏 Rail / 窄屏 BottomBar）：

| Tab | 职责 |
|-----|------|
| **书架** | 继续阅读、最近、我的书架（钉选）；可选展示钉选合集（中） |
| **书库** | 全部单本；搜索 / 筛选 / 排序 / 管理；**书单**与**合集**为二级（见 §4.4a） |
| **设置** | 本品牌外观、本引擎阅读默认、关于 |

- **搜索**、**书单**、**合集** 宿主在书库侧（或书架辅区），**不是**第四根 Tab。  
- 书库内 **没有**「漫画 | 图书」分段（每个 App 本身已是单一品类）。

### 4.4a 整理三概念（权威划分）

同一 App 内三种整理方式，**心智与 UI 不得混用**：

| 概念 | 用户说法 | 用途 | 主形态 | 状态 |
|------|----------|------|--------|------|
| **我的书架** | 上架 / 钉选 | 想常看见的固定池（全局一个） | 书架横滑单本封面 | **已有** |
| **书单** | 书单 | **长清单、按序读**（待读队列、跟读） | 目录=文字列表；内容=竖向长列表 | **已有**（comic；UI 保持列表感） |
| **合集** | 合集 / 分组 | **系列/主题收纳**（一盒） | **书库**封面尺寸拼贴卡（与单本同网格、合集排最前） | **已有**（comic） |

- **书单 ≠ 合集**：书单不为拼贴大卡；合集不做「几十行待读队列」主路径。  
- 单本可同时：上架 + 进多个书单 +（规划）进一个主合集。  
- 交互细则：[shelf.md](./specs/shelf.md) · [lists.md](./specs/lists.md) · [collections.md](./specs/collections.md)。

## 5. 用户路径（每个 App 各自）

1. **首次**：导入本品类文件 → 书库 → 打开阅读 → 书架继续读  
2. **续读**：书架 → 恢复进度  
3. **查找**：书库搜索 / 排序 → 打开  
4. **整理**：删除、上架；书单（队列）；合集（系列盒，中）  

用户若两种都要：安装 **两个 App**，不是一个 App 里切换模式。

---

## 6. 条目生命周期（单 App 内）

```text
导入（本 App 白名单 + hash）→ 书库
  → 打开 → lastOpenedAt + progress → 书架继续/最近
  → 上架 → 我的书架（钉选）
  → 书单 → 清单成员（长列表）
  → 合集 → 合集成员（拼贴盒，中）
  → 删除 → 级联清理进度/成员关系
```

两 App **数据默认不互通**。

## 8. 格式矩阵

| 格式 | 默认归属 App | 引擎 | 状态 |
|------|----------------|------|------|
| CBZ / ZIP | comic | 页图 | **已有** |
| EPUB（页图 / 固定版式漫画） | **comic** | 页图（OPF spine → 图） | **已有** |
| EPUB（reflow 正文） | book | reflow | **中**（book 阶段） |
| TXT / MD | book | reflow | **中～远** |
| PDF | 待定 | 待定 | **远** |
| MOBI | book | 待定 | **远** |

**comic 的 EPUB**：按图片页阅读（spine 抽图）；纯文字 reflow EPUB 会导入失败并提示。  
**book 的 EPUB**：留给 reflow 引擎，不与 comic 页图路径混用同一阅读器。

## 9. 非目标

- 单 App 统一库混装漫画+图书作为主路径  
- 双 App 默认共享阅读进度 / 同一数据库  
- 社区、商城、强制云刮削  
- 迁移旧 Swift 数据  
- 用**书单** UI 冒充**合集**拼贴（或反过来）  
- 第四个根 Tab 专放书单/合集
