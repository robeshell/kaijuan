# Kaika 设计总纲

这份文档定义 KaikaNext 的视觉与交互方向。实现细节可以演进，但偏离这些原则需要明确的设计评审。

专项规范（随功能落地逐步补齐，先写 spec 再写 UI）：

- 阅读器外观 → `specs/reader-chrome.md`（Phase 1 漫画阅读器开工前）
- 设置信息架构 → `specs/settings.md`
- 浮层 / 菜单 / 弹窗 → `specs/overlay-menu.md`
- 手机 / 紧凑布局覆盖规则 → `specs/mobile.md`

## 产品性格

Kaika 是一个多格式个人阅读器（图书 + 漫画）。它应该让人感到**安静、干净、像一间采光好的书房**：内容（封面、书页、漫画）提供色彩，外壳保持中性克制，不喧宾夺主。

## 批准的视觉方向

- **浅色模式是产品默认和首要验收目标**；深色模式由同一套语义 token 支持，但不是验收基线。
- **轻玻璃中性风**：默认画布是中性近白，不偏暖不偏蓝。
- 侧栏、导航、阅读器 chrome、菜单、对话框使用**半透明磨砂玻璃**；普通列表行和卡片使用更廉价的半透明填充，避免长滚动面上叠加模糊。
- 玻璃保持克制：不要科幻蓝 tint、不要强辉光、不要镜面 chrome、不要牺牲文字对比度的装饰性透明。
- **内容提供色彩**：封面和书页是画面的颜色来源，chrome 保持中性，不与封面争色。
- 强调色只承担：当前态 / 选中 / 确认动作 / 关键进度。同一区域最多一个主强调元素。
- macOS 使用透明全尺寸标题栏：背景延伸到原生红绿灯后面，隐藏重复窗口标题，交互内容从 38px 安全区下方开始。

## 外观分层

外观由独立层组合，而不是一个单体主题：

- **主题模式**：浅色 / 深色 / 跟随系统。浅色是验收基线。
- **强调色**：控制选中、进度、焦点、确认动作。默认暖橙（延续 kaika 的阅读器血统），其余预设同一 token 体系内可换。
- **阅读主题（独立体系）**：阅读器**内容区**不属于 chrome，使用专门的阅读主题（纸白 / 米色 / 深灰 / 纯黑），跟随书籍场景而非 App chrome。阅读器的工具栏、面板、弹窗仍走 chrome 的玻璃语言。

外观选择本地持久化、即时预览、选择器不折叠。

## 语义 token 三层模型

代码结构固定为三层，业务代码只允许引用第 2、3 层：

1. **原始 token**：色板、间距刻度、圆角刻度、字阶（`lib/core/kaika_theme.dart`）
2. **语义 token**：`canvas / surface / glassFill / hairline / textPrimary / textSecondary / accent`
3. **主题组装**：亮/暗 × 强调色 → `ThemeData`

禁止业务 UI 直接写死 `Color(0xFF…)` 或 `Colors.white` 这类原始值。

## 核心屏幕原则

1. **书架**：继续阅读优先，封面是主角；空态要安静，不要插画堆砌。
2. **书库**：书籍 / 漫画双分段；卡片大封面 + 紧凑标题；来源、格式等信息收进筛选和详情，不堆在每张卡片上。
3. **设置**：页面标题 + 水平分类锚点 + 单一受限表单列；桌面端文字优先的安静行样式，紧凑端分组索引 + 底部弹层选择器。
4. **阅读器**：chrome（顶栏/底栏/面板）用玻璃语言，默认自动隐藏；内容区用阅读主题。弹层、tip、注释编辑器必须从同一套面板语言延展，不回退到平台默认 Material 表单风格。

## 间距与形状基调

- 间距刻度：4 / 8 / 12 / 16 / 24 / 32，组件内外边距只取刻度值。
- 圆角刻度：8（小控件）/ 12（卡片、按钮）/ 16（内嵌卡片）/ 20（面板、弹窗）。
- 描边即分隔：优先用 `hairline`（主文字色 8% 透明度）而非实色分隔线。
- 阴影要柔和低位，像纸浮起，不像悬浮广告卡片。
