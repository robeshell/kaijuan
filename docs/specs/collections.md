# 合集（分组）设计规范

| **状态** | **已有**（comic）：表 + 拼贴卡 + **书库**最前混排 |

## 目标

本 App 内把多本条目收成 **视觉上的一盒**——为「系列 / 主题收纳」服务。  
外形接近 **单本封面尺寸的大卡**，内部排列 **多枚小封面**（合集感）。  

**不是**书单长列表（见 [lists.md](./lists.md)）；**不是**我的书架钉选（见 [shelf.md](./shelf.md)）。

## 用户心智

| 对 | 不对 |
|----|------|
| 「这一套是同一作品多卷」「科幻主题一盒」 | 「按顺序读完这个队列」→ 书单 |
| 在书库里像「一本」一样扫视 | 打开后是几十行待读清单 |

## 命名

| 层 | 用词 |
|----|------|
| 用户可见（推荐） | **合集** |
| 文档/代码可选 | collection；内部可称 group |
| 避免 | 与「书单」「书架」混叫「收藏夹」 |

## 信息架构

```text
书库主网格/列表
  · 【合集卡…】排在最前（与单本同尺寸）
  · 【未入集的单本…】在后
  · 已加入合集的单本 **不在书库主列表重复出现**（仅在合集内可见）

书库 · 二级「合集」
  · 合集目录管理（新建/重命名/删除）

详情 / 多选 → 加入合集
```

- **非根 Tab**；**不在书架「我的书架」展示**（书架仅单本钉选 + 继续阅读）。  
- 合集成员来自同一 `app_library` 的漫画与图书条目。

## 展示形态（核心差异）

### 合集卡（目录 / 混排）

- 外框尺寸 **约等于** 书库单本封面卡。  
- 内部：常见 **2×2 小封面**，或 1 大 + 若干小；不足用占位。  
- 标题：合集名；副文：`N 本`。  
- **禁止**把书单目录做成这种卡（书单目录保持文字列表）。

### 合集内容页

- 以 **网格** 浏览组内条目为主（与书库单本卡一致语言）。  
- 可提供列表切换（**中**）。  
- 点条目 → 本 App 阅读器。

## 数据（规划）

- 新表（示例）：`collections` + `collection_members`。  
- 与 `reading_lists` **分开**，不要复用书单表硬撑 UI。  
- v1 建议：**一本书最多归属一个主合集**（降低心智；多合集 **远**）。  
- `seriesName` 元数据可作弱提示，**不**自动强制建合集（v1）。  
- 删合集不删条目；删条目移出成员。

## 操作（v1 范围）

| 操作 | 说明 |
|------|------|
| 新建 / 重命名 / 删除合集 | 合集目录或详情 |
| 加入 / 移出 | 条目详情、多选「加入合集」 |
| 打开合集 | 点合集卡 |
| 打开成员 | 合集内容页点封面 |

可选 **中**：指定「代表封面」、组内展示序、钉到书架。

## 空态

- 无合集：引导「把系列收成一盒」。  
- 合集内空：引导从书库加入。

## 视觉

- 合集内容页多选底栏与书库共用 `SelectionActionSheet`（图标网格，不单行塞按钮）。

## 与书单对照（实现时自检）

| | 书单 | 合集 |
|--|------|------|
| 目录 UI | 文字行列表 | 封面尺寸拼贴卡 |
| 内容 UI | 竖向长列表 | 网格为主 |
| 主任务 | 按序读 | 收纳与浏览 |
| 表 | `reading_lists*` | `collections*`（待建） |

## 非目标（v1）

- 用合集代替书单队列。  
- 网络分享合集。  
- 文件夹导入自动建合集（可 **中～远**）。  
- 跨品牌合集。
