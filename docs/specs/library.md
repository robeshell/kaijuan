## 目标

**当前 App** 内已导入条目的浏览与管理。  
comic App：页图类（CBZ/ZIP + **页图式 EPUB**）；book App：reflow 图书——**不做**库内「漫画 | 图书」分段。

## 范围

- **做**：网格/列表、导入、打开（本引擎）、搜索/排序/筛选、详情、多选、**书单**二级、（规划）**合集**。  
- **不做**：跳转到另一产品 App；混装另一品类作主路径。

## 信息架构

```text
书库
  · 主区：单本（网格 / 列表）
  · 过滤器：状态 / 上架 / 格式 / 搜索 / 排序
  · 二级入口：
      - 书单 → 长清单管理（lists.md）
      - 合集 → 拼贴盒管理（collections.md，中）
  · 合集卡（中）：可与单本同网格混排，或仅出现在合集目录
```

- 导入扩展名 = **本 BrandConfig 白名单**。  
- **书单**与**合集**入口文案、图标必须区分（列表 vs 拼贴盒）。

## 打开路由

```text
点单本 → 本 App 引擎打开
  comic app → 漫画阅读器
  book app  → 图书阅读器（未就绪：明确暂不支持）

点合集卡 → 合集内容页 → 再点成员 → 引擎
点书单 → 书单内容列表 → 再点成员 → 引擎
```

## 卡片 / 空态 / 视觉

- **单本卡**：封面 + 两行标题；可选进度 / 格式角标。  
- **合集卡**（中）：单本卡同尺寸外框 + 内部小封面拼贴；见 collections.md。  
- 空态文案按品牌（「导入漫画」vs「导入图书」），主按钮「导入」。  
- Token 与间距见 DESIGN_FOUNDATION；强调色来自品牌配置。

## 多选

- 批量：删除、上架/移出书架、加入书单；（中）加入合集。  
- 不在多选里混淆「创建书单」与「创建合集」的主按钮文案。
