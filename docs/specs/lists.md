## 目标

App 内 **可命名、可很长的阅读清单**——为「按单子一本本看」服务。  
**不是**合集拼贴盒子（见 [collections.md](./collections.md)）；**不是**我的书架钉选（见 [shelf.md](./shelf.md)）。

## 用户心智

| 对 | 不对 |
|----|------|
| 「周末待读」「连载跟读队列」 | 「把系列收成一盒封面」→ 合集 |
| 顺着列表往下读 | 在书库网格里用大卡展示组 |

## 信息架构

- **非根 Tab**；宿主在 **书库** 二级（顶栏「书单」入口）。  
- 书单 **目录页**：文字列表（名称、本数、最近更新）— **不要**做成合集拼贴网格。  
- 书单 **内容页**：**竖向长列表**（小封面 + 标题 + 进度）；支持滚动很多条目。  
- 书单成员来自同一 `app_library` 的漫画与图书条目。

## 数据（实现对照）

- 表：`reading_lists` + `reading_list_members`（已有）。  
- 成员宜支持 **顺序**（`sortOrder` / `addedAt`；后续可拖拽排序 — **中**）。  
- 删书单不删条目；删条目级联移出成员。

## 操作

| 操作 | 入口 |
|------|------|
| 新建 / 重命名 / 删除书单 | 书单目录页 |
| 加入书单 | 条目详情、书库多选 |
| 移出书单 | 书单内容页（长按/菜单） |
| 打开阅读 | 内容页点条目 → 本 App 引擎 |

## 空态

- 目录空：「还没有书单」+ 新建；说明可从详情/多选加入。  
- 内容空：「书单为空」+ 引导回书库。

## 视觉

- 列表感、安静；小封面，不抢单本网格。  
- 图标：playlist / 列表语义；**不要**用多封面叠盒（留给合集）。  
- Token 见 DESIGN_FOUNDATION。

## 非目标

- 网络分享书单。  
- 用书单 UI 冒充合集拼贴卡。  
- 与「我的书架」合并成一个实体。
